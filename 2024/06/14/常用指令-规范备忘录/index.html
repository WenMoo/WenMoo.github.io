常用指令/规范备忘录

Author: Wen Mo Date: June 14, 2024  13:47:04

SVN 常用基本命令

  1. 从服务器端下载代码到本地

    • svn checkout 服务器地址 Users/apple/Documents/code
    • 指令的意思：将服务器中repository仓库的内容下载到/Users/apple/Documents/code目录中

  2. 更新服务器端代码到本地目录

    • 进入到目标目录
    • svn update
    • 如果需要切换到指定节点就在后面加上 svn update -r 节点序号

  3. 提交代码 （之前表不存在的，新添加的）

    • 进入需要提交的目录
    • svn state 查看状态
    • svn add 文件
    • svn commit -m “修改了XXX文件”
    • 这个命令会将目标目录下的所有修改都提交到服务器端

  4. 提交更改过的代码 （之前表存在的，修改的）

    • 进入需要提交的目录
    • svn state 查看状态 ，能看到是有个 M 的
    • svn commit -m “修改了XXX文件” path
    • 这个命令会将目标文件提交上去

  5. 解决冲突

    • 进入目标SVN文件夹
    • svn revert 冲突相对位置
    • svn up

  6. 账号更新

    • 通过 svn auth 可以看到当前账号信息地址
    • 进入文件夹中删除 auth 这个文件夹
    • 钥匙串-登录中找到关于 svn 的，修改为需要的账号密码
    • 这会我们正常svn update 就会重新填入了

Git 提交规范

feat: A new feature
壮举：新功能

fix: A bug fix
修复：错误修复

docs: Documentation only changes
docs：仅文档更改

style: Changes that do not affect the meaning of the
样式：不影响代码含义的更改（空格、格式、缺少分号等

refactor: A code change that neither fixes a bug nor
重构：既不修复错误也不添加功能的代码更改

perf: A code change that improves performance
perf：提高性能的代码更改

test: Adding missing tests or correcting existing tests
测试：添加缺失的测试或纠正现有的测试

build: Changes that affect the build system or external dependencies
build：影响构建系统或外部依赖项的更改

ci: Changes to our CI configuration files and scripts
ci：对我们的 CI 配置文件和脚本的更改

chore: Other changes that dont modify src or test
杂项：不修改 src 或测试文件的其他更改

revert: Reverts a previous commit
还原：还原以前的提交

我的基本命名规范

1、元素名/属性名

1.基本结构，考虑为元素命名其本身的作用或”用意”，达到语义化。不要使用表面形式的命名，命名的方式能描述出正在做的事情。

  • [元素作用/内容]-[元素类型]
  • 如:red/left/big等，而是比如一个搜索的按键：searchBtn ，

2.补充元素范围，因为我们一个页面可能包含多个按键，甚至说都是一样功能的，我们就需要对其区分，可以给他添加上范围。

  • [元素范围]-[元素作用/内容]-[元素类型]
  • 如:搜索按钮: tabelSearchBtn ，登录表单:topLoginBtn，新闻列表下一页: leftNextPageBtn

2、事件 / 函数命名

核心原则是：函数名 = 函数做了什么

所以一般能分为四类：

  • on ： 事件入口
  • handle: 业务处理
  • update: 状态更新
  • refresh: 全量刷新

所以比如上面搜索按钮的触发事件的生命周期可以有：

// UI 层

onSearchBtnClick()

// 业务层

handleSearch()

// 状态层

updateSearchResult()

// 回调

onSearchFinished()

3、文件的命名

  • 驼峰命名+大写开头
